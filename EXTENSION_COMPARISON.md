## 🎯 使用场景推荐

### 使用 zcy-goldminer (基础版) 的情况：
- ✅ 你已经知道确切的类目ID

- ✅ 需要探索更多可能的类目
- ✅ 想要一键操作，减少手动工作
- ✅ 需要定期更新类目数据
- ✅ 要实现最大覆盖率（120%）

**示例**：你想建立一个全品类的SaaS系统，需要尽可能多的类目数据。

## 📊 工作原理对比

### zcy-goldminer 工作流程
```
用户打开页面
    ↓
扩展注入UI面板
    ↓
用户点击"开始挖掘"
    ↓
根据硬编码的种子ID
    ↓
递归调用API
    ↓
构建类目树
    ↓
导出JSON
```

**优点**：逻辑清晰，可控性强  
**缺点**：需要预先知道ID，无法发现新类目

### zcy-auto-miner 工作流程
```
用户打开页面
    ↓
注入 inject.js (拦截器) + content.js (控制器)
    ↓
用户点击"开始自动挖掘"
    ↓
扫描左侧菜单 → 关键词匹配
    ↓
自动点击匹配项（间隔1.5秒）
    ↓
inject.js 实时拦截网络请求
    ↓
提取类目数据 → 发送给 content.js
    ↓
去重存储
    ↓
导出JSON
```

**优点**：全自动，覆盖面广，可发现新类目  
**缺点**：依赖页面结构稳定性

## 🔧 技术实现对比

### zcy-goldminer
- **核心技术**：直接 Fetch API 调用
- **数据获取**：主动请求后端接口
- **递归深度**：3层（可配置）
- **种子数据**：硬编码数组

```javascript
const SEED_IDS = [
    { name: "计算机设备及软件", id: 3564 },
    { name: "3C数码", id: 4400 },
    // ...
];
```

### zcy-auto-miner
- **核心技术**：
  - Content Script（DOM控制）
  - Injected Script（API拦截）
  - PostMessage（跨环境通信）
- **数据获取**：被动拦截页面请求
- **覆盖范围**：基于关键词动态匹配
- **关键词库**：可扩展数组

```javascript
const TARGET_KEYWORDS = [
    "计算机", "办公设备", "3C", "数码",
    // 21+ 关键词...
];
```

## 🚀 推荐使用策略

### 策略一：单独使用 auto-miner
如果你追求最大覆盖和自动化：
1. 安装 `zcy-auto-miner`
2. 运行自动挖掘
3. 如需补充，运行 `gap-fill-script.js`
4. 完成！

### 策略二：组合使用
如果你想兼顾准确性和覆盖面：
1. 先用 `zcy-auto-miner` 获取大部分数据
2. 对于核心类目，用 `zcy-goldminer` 进行精确递归挖掘
3. 合并两份 JSON 数据

### 策略三：分场景使用
- **日常更新**：用 `zcy-auto-miner`（快速全面）
- **深度挖掘**：用 `zcy-goldminer`（精确控制）
